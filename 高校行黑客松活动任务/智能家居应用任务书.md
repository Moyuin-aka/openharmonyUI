# 鸿蒙智能家居控制应用开发 AI编程挑战赛需求文档

### 任务一：应用内首页智能设备卡片组件效果实现

![**参考效果图**](C:\Users\Administrator\Desktop\新建文件夹\设备卡片组件.png)

**开发需求**

1. 卡片布局实现：完成参考效果图中带圆角、毛玻璃半透明效果的设备卡片开发，还原卡片阴影、内边距、元素分区布局，与效果图视觉效果一致。
2. 设备元素展示：单张卡片需完整包含设备图标、设备名称、在线状态标签、设备运行参数（亮度/温度/开合度/电量等）、控制按钮五大核心元素，各元素排版与效果图完全匹配。
3. 交互功能实现：控制按钮支持点击交互，点击后同步更新设备状态与参数，同时弹出吐司提示（如“客厅主灯状态已更新”），3秒后自动消失；在线/离线状态对应不同的按钮可用状态，离线状态下按钮禁用。
4. 文字样式规范：主标题（设备名称）、副标题（状态/参数）的字体、字号、字重、色值严格匹配效果图，文字超出长度做截断处理。
5. 组件要求：设备卡片组件独立封装，支持设备数据、样式的自定义配置与外部数据绑定，可被全局继承复用。

**开发工具要求**
使用DevEco Studio和其他AI工具（DevEco CodeGenie/Code Arkts/Trae等）辅助进行代码开发。

### 任务二：鸿蒙元服务桌面卡片效果实现

![服务卡片桌面效果](C:\Users\Administrator\Desktop\新建文件夹\服务卡片桌面效果.png)

**开发需求**

1. 视觉效果还原：开发HarmonyOS元服务桌面卡片，实现与任务一应用内卡片1:1的视觉效果，包括圆角、毛玻璃背景、元素排版、文字样式完全一致。
2. 核心功能实现：桌面卡片完整展示设备核心信息（设备图标、名称、在线状态、运行参数），支持2x2、2x4两种常用卡片规格适配。
3. 交互跳转能力：点击桌面卡片可直接跳转至应用内对应的设备控制页面，点击卡片内控制区域可触发设备状态更新。设备控制页面可以自行发挥创新，不做要求。
4. 复用性要求：可直接复用任务一卡片组件的样式代码、数据结构，无需重复开发核心逻辑。

**开发工具要求**
使用DevEco Studio和其他AI工具（DevEco CodeGenie/Code Arkts/Trae等）辅助进行代码开发。

### 任务三：智能设备列表响应式布局与滚动容器实现

![手机竖屏效果](C:\Users\Administrator\Desktop\新建文件夹\手机竖屏效果.png)

![个人智能助手效果图](C:\Users\Administrator\Desktop\新建文件夹\个人智能助手效果图.png)

**开发需求**

1. 滚动容器开发：使用任务一开发的设备卡片组件，完成设备列表滚动容器开发，支持列表上下无限滚动，滚动过程无卡顿、无渲染异常。
2. 响应式布局实现：核心实现横竖屏切换自适应能力——竖屏状态下1行展示1个设备卡片，横屏状态下1行展示2个设备卡片，切换过程无布局错乱、元素挤压。
3. 动态渲染能力：列表支持通过数据动态渲染设备卡片，卡片内容不重复，支持下拉刷新更新设备状态与列表内容。
4. 多尺寸适配：适配手机、平板、折叠屏等不同尺寸的鸿蒙设备，所有设备下布局均符合自适应规则，无显示异常。

**开发工具要求**
使用DevEco Studio和其他AI工具（DevEco CodeGenie/Code Arkts/Trae等）辅助进行代码开发。

### 任务四：Mock数据模拟与跨端数据联动实现

![mock](C:\Users\Administrator\Desktop\新建文件夹\mock.png)

**开发需求**

1. Mock数据工程搭建：在应用工程的`mock`目录下创建模拟数据文件，定义标准化的设备数据结构，包含设备ID、设备类型、设备名称、在线状态、运行参数、图标资源等核心字段。
2. 多设备类型模拟：基于mock数据，模拟至少6种智能家居设备类型，包括但不限于客厅主灯、卧室空调、客厅窗帘、扫地机器人、智能门锁、加湿器，覆盖效果图内的全部设备类型。
3. 数据联动同步：实现应用内设备卡片与任务二开发的桌面元服务卡片的数据实时联动——在应用内修改设备状态/参数后，桌面卡片同步更新对应数据；mock数据更新后，应用内列表与桌面卡片同步刷新。
4. 动态扩展能力：mock数据支持动态增删设备条目，新增设备可自动渲染至应用内列表，同时支持同步至桌面卡片展示。

**开发工具要求**
使用DevEco Studio和其他AI工具（DevEco CodeGenie/Code Arkts/Trae等）辅助进行代码开发。
