# 一、评价准则

### 核心评价维度及权重

| 评价维度    | 权重占比 | 核心考察目标                        |
| ------- | ---- | ----------------------------- |
| 功能完整性   | 30%  | 需求功能是否100%实现，组件复用性与数据联动是否符合预期 |
| UI还原度   | 25%  | 视觉效果与参考效果图的匹配程度，跨场景视觉一致性      |
| 代码质量    | 20%  | 代码结构、组件封装复用性及AI工具运用合理性        |
| 适配与交互体验 | 15%  | 横竖屏适配效果、操作流畅度与反馈合理性           |
| 开发效率    | 10%  | 任务完成速度与时间把控能力                 |

### 具体评分标准

#### 功能完整性(30%)

| 评分项                 | 评分细则                                                                                                       | 分值区间  |
| ------------------- | ---------------------------------------------------------------------------------------------------------- | ----- |
| 任务一：应用内首页智能设备卡片组件功能 | 设备核心元素完整展示(图标/名称/在线状态/运行参数,5%)；控制按钮点击交互+吐司提示(4%)；卡片毛玻璃/圆角/布局效果还原(5%)；主副标题/状态文字样式符合设计(3%)；支持外部数据绑定与组件复用(3%) | 0-20% |
| 任务二：鸿蒙元服务桌面卡片功能     | 与应用内卡片视觉效果1:1还原(2%)；设备状态与核心参数完整展示(2%)；点击卡片可跳转至应用对应页面(2%)；支持复用任务一的样式与数据结构(1%)                               | 0-7%  |
| 任务三：智能设备列表响应式布局功能   | 横竖屏切换布局自适应生效(3%)；竖屏1行1个/横屏1行2个的网格布局正确(2%)；列表上下滚动流畅无卡顿(1%)；支持下拉刷新设备状态(1%)；动态数据渲染无重复(1%)                     | 0-8%  |
| 任务四：Mock数据模拟与数据联动功能 | mock目录下设备模拟数据结构完整(2%)；支持≥6种设备类型模拟(1%)；应用内卡片与桌面服务卡片数据实时联动同步(2%)                                             | 0-5%  |

#### UI还原度(25%)

| 评分项     | 评分细则                                    | 分值区间  |
| ------- | --------------------------------------- | ----- |
| 核心视觉还原度 | 卡片圆角、毛玻璃半透明效果、背景色、阴影与参考效果图100%匹配，无视觉偏差  | 0-10% |
| 元素排版一致性 | 设备图标、文字、按钮的间距、对齐方式、尺寸比例完全符合效果图，无元素错位、溢出 | 0-8%  |
| 文字样式规范  | 字体、字号、字重、色值与效果图完全匹配，状态文字（在线/离线）色值区分符合设计 | 0-4%  |
| 跨场景视觉统一 | 应用内卡片与桌面元服务卡片视觉效果完全一致，无样式差异             | 0-3%  |

#### 代码质量(20%)

| 评分项       | 评分细则                                                            | 分值区间  |
| --------- | --------------------------------------------------------------- | ----- |
| 封装与复用性    | 设备卡片组件独立封装可继承(5%)；样式、数据、逻辑模块化拆分(3%)；冗余代码≤10%(2%)                | 0-10% |
| AI工具运用合理性 | 基于AI生成代码完成业务适配与样式优化(5%)；变量/组件/方法命名语义化规范(3%)；无明显AI生成逻辑漏洞与死代码(2%) | 0-10% |

#### 适配与交互体验(15%)

| 评分项     | 评分细则                                                    | 分值区间 |
| ------- | ------------------------------------------------------- | ---- |
| 适配完整性   | 横竖屏切换无布局错乱、元素挤压；不同尺寸设备下自适应效果正常(5%)；折叠屏适配无异常(3%)         | 0-8% |
| 交互反馈完整性 | 按钮点击即时视觉反馈(颜色/大小变化)、功能触发吐司提示(4%)；设备状态切换、列表滚动动画流畅无掉帧(3%) | 0-7% |

#### 开发效率(10%)

| 评分项    | 评分细则                                                           | 分值区间  |
| ------ | -------------------------------------------------------------- | ----- |
| 时间把控能力 | 2小时内完成全部需求(6%)；提前30分钟及以上且核心功能完整(+4%)；超时10分钟内(-3%)；超时10分钟以上(0%) | 0-10% |

### 扣分红线

| 严重问题类型                                    | 处理规则             |
| ----------------------------------------- | ---------------- |
| AI工具使用率不足30%                              | 直接判定为不合格(< 60 分) |
| 核心功能未实现(如卡片无法渲染、控制按钮无响应、横竖屏切换布局错乱、数据联动失效) | 直接判定为不合格(< 60 分) |
| 运行时崩溃、白屏、页面卡死等严重问题                        | 直接判定为不合格(< 60 分) |

### 整体说明

1. 总分100分，各维度得分累加计算，划分为优秀(90+)、良好(80-89)、合格(60-79)、不合格(<60)。
2. 同一问题不重复扣分，以最高扣分值为准。
3. 评分时需结合参考效果图和功能清单逐项核查。
